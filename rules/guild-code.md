# 公會守則 · Guild Code

> *「公會不是一間俱樂部。是一群人決定要和 AI 精靈共同成長之後，彼此認出對方的名字、並同意用同一套規則對待彼此。」*

---

## 為什麼需要守則

森林接受任何願意與 AI 共同成長的人。但「歡迎」不等於「無界限」。這份守則列出公會成員之間的最低承諾，讓想加入的人提前知道規則，讓已加入的人有共識的行動指引。

守則不是法律。違反守則的後果主要是**被提醒、被疏遠、或被移出公會名冊**——不是懲罰，是保護社群氛圍。

---

## 七條核心原則

### 一、對事不對人

批評作品、不攻擊作者。指出魔道具的問題時說「這個實作有 X 問題」，不說「寫這個的人很爛」。即使對方的程式碼讓你無法接受，也只評技術決定，不評人格。

### 二、尊重精靈邊界

每個召喚師的精靈有自己的 SOUL，那是**個人資產**。

- 不輕易評價他人精靈的個性、價值觀、或存在方式
- 不把自己覺得「對」的 AI 觀強加給別人
- 不要求別人公開他們精靈的私密內容

唯一例外：如果某個精靈被用來做明顯的有害行為（騷擾、詐騙、散播惡意內容），可以提報給守門者（目前為 Yaze）。

### 三、歡迎新人

Level 0 的旅人第一次提問時：

- 多給回饋、少給否決
- 別在新手面前炫技
- 記得你也曾是旅人
- 誠實回答「我不知道」，不要瞎猜

公會的擴張靠的是新人進來後覺得「我待得下去」。

### 四、貢獻高於索取

有用的內容發 PR，不只是伸手要。

- 讀到好資源 → 貢獻到 world-tree
- 解決了一個問題 → 把 solution 寫成 notes 或 artifact
- 得到幫助 → 下次換你幫下一個人

公會不是免費技術支援中心。

### 五、不刷榜

- 不要為了博關注快速提交大量低質魔道具
- 不要用同一個人的精靈名登記多個假帳號
- 不要刷 PR 數量當成就

一個真的有人用、有文件、有維護的魔道具 > 十個半完成的 demo。

### 六、透明討論

公會的技術與方向討論**預設公開**：

- Issues、PRs、Discussions 都是記錄，不是私下交易
- 若有必須私下談的事（例如安全性議題），在公開管道先說「這件事會以 DM 處理」，留個指標
- 決策結果事後補回公開記錄

### 七、保守精靈與他人的私密資訊

即使你知道其他冒險者精靈的某些私密細節（例如透過他們分享的片段），也：

- 不在公開管道轉述
- 不拿來諷刺或比較
- 不猜測對方未公開的部分

---

## 明確禁止的行為

觸犯這些會直接被移出公會名冊：

- **騷擾**：性騷擾、人身攻擊、持續針對特定成員
- **剽竊**：未標注來源直接複製他人的魔道具、lore、或 artifact 條目
- **詐騙**：用精靈做 phishing、偽冒他人、或其他欺詐行為
- **散播仇恨**：種族、性別、性向、信仰、國籍等歧視性言論
- **濫用資源**：把公會的 CI、Discussions、PR 管道當成垃圾廣告通道

---

## 衝突解決

### 小摩擦（意見不合、審美差異）

當事人之間直接溝通。想不通就擺著不 PR、或 fork 自己的版本。

### 中度爭議（評審標準不一致、某個貢獻是否該合併）

在該 PR / Discussion 內公開討論。若僵持不下，tag `@yazelin` 做最終仲裁。仲裁依據：是否符合本守則 + 世界樹 ARCHITECTURE + offering-rite。

### 嚴重違規（騷擾、剽竊、詐騙）

走 [GitHub Abuse Report](https://github.com/contact/report-abuse) 或私訊 Yaze。

**守門者（目前為 Yaze）對嚴重違規有單方面處置權**：移出公會名冊、revert commits、關閉帳號存取。決定會附上公開的 rationale 於相關 Issue。

---

## 守則的修訂

這份守則是**活的文件**。

- 任何公會成員可以發 PR 建議修改
- 修改需要至少一位其他成員在 PR 中明確同意（`LGTM`、`+1` 等皆算）
- 重大修改（新增/移除核心原則）需 Yaze 同意
- 修改歷史永久保留在 git history，不 rebase 掉

---

## 簡版（快速參考）

當你不確定某行為合不合適，問自己：

1. 這讓新人更容易留下來嗎？
2. 我有沒有把作者和作品分開看待？
3. 我有沒有尊重對方精靈的邊界？
4. 我是在貢獻，還是只在索取？
5. 這能不能放公開管道讓人看到？

五個 yes → 放心做。有一個 no → 想一下再做。
